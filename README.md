# Project LoreBoard (临时命名)
**—— 基于《密教模拟器》隐喻的 AI 桌面编排控制台**

## 1. 核心理念 (Core Philosophy)
LoreBoard 抛弃了传统 AI Dashboard 枯燥的“终端 Log”和“静态表单”交互，将 AI 代理的异步调度、上下文依赖、资源消耗和异常处理，转化为极具物理感和仪式感的“桌面卡牌合成与状态流转”体验。

在这个控制台上：
- **资源不再是文件，而是带有标签和倒计时的「实体卡牌」。**
- **任务不再是按钮，而是吞噬和吐出卡牌的「动词插槽」。**
- **报错不再是控制台红字，而是会污染你桌面的「副作用卡牌」。**

## 2. 界面隐喻映射 (UI Metaphor Mapping)

| 传统 AI 概念 | LoreBoard 隐喻 | 表现形式 |
|------------|--------------|---------|
| 资源库/文件管理器 | **抽屉 (Drawer/Hand)** | 底部存放可用资源（密钥、长文本、数据表） |
| 任务模板/执行脚本 | **动词 (Verb/Contract)** | 左侧的常驻功能块（研究、提取、执行、复盘） |
| 工作区/日志看板 | **桌面 (Tabletop/Canvas)** | 中央的二维拓扑画板，资源在这里被消耗、加工、掉落 |
| Input 校验/参数 | **插槽 (Slot)** | 动词展开后的占位符，仅接受特定 Tag 的卡牌 |
| 异步等待/队列 | **倒计时/转圈** | 放入卡牌并点火后，动词进入物理转圈倒计时状态 |
