# 核心机制与玩法 (Core Gameplay Mechanics)

## 1. 实体卡牌与属性 (Entity Cards)

所有的资源（无论物理文件还是抽象授权）都将抽象为实体卡牌。卡牌不仅有类型，更有**生命周期**和**消耗方式**。

- **动态源 (Dynamic)**：如搜索引擎调用权限（如 Web Search API）、RSS Feed 连接。这些卡片像“水龙头”，放进插槽后可以持续产出新资源，或者每次请求产生一次计算开销。
- **静态文档 (Static)**：纯文本、Markdown、图片、PDF 等可以被分析和提纯的“材料”。
- **凭证与密钥 (Secrets)**：特殊的红色/金色卡牌。它们是驱动高阶任务（如爬取付费墙）的“钥匙”，必须被放入指定的【鉴权插槽】才能使动词（任务）点火。

## 2. ⏳ 卡牌衰减与临时授权 (Decay & Ephemeral Resources)

这是对“过期状态”的极客隐喻。
- **倒计时卡牌**：一个通过 OAuth 获取的只有 1 小时有效期的临时 Token，或者是刚抓取下但尚未被提纯的网页源码（如果不抓紧处理就会变灰丢失）。
- **交互**：卡牌上显示沙漏动画。玩家（调度者）必须在它失效前将其拖入任务插槽（动词）中“用掉”。这比在控制台里看一堆过期的 Error Logs 直观且刺激得多。

## 3. ⚠️ 副作用与异常卡牌 (Errors as Entities)

传统监控是看 Log 报错，LoreBoard 则是让报错“实体化”，甚至会污染你的控制台桌面。
- **红色异常卡牌（如：Rate Limit / CAPTCHA 拦截）**：如果你的爬虫代理访问过频，动词吐出的不仅有残缺的数据卡牌，还会掉落一张刺眼的红色警告卡。
- **灾难事件 (Nemesis Verb)**：如果你桌面上积累了太多 `[报错卡]` 没处理，系统会自动生成一个强制执行的“猎人/封禁”动词。它会像磁铁一样把你的正常 API 密钥卡牌吸走（代表账号被封禁）。
- **处理方式**：你需要用 `[切换 IP 代理]` 或 `[人工介入干预]` 的卡牌，放入这个警告插槽里，才能将其抵消（消除异常）。

## 4. 🧲 磁吸与计算消耗机制 (Magnet Slots & Token Burn)

把后台持续跑着的守护进程（Daemon Agent）或者轮询任务做成带有呼吸灯的动词块。
- **持续吸血**：每隔一个周期，这个常驻监控动词就会自动从你的资源抽屉或桌面上，吸走一张 `[API Token 额度]` 或 `[算力点数]` 卡牌。
- **饥饿状态**：如果你忘了补充“口粮”（比如 OpenAI 余额告警），常驻 Agent 的动词就会变成饥饿状态（灰暗、停转），必须喂入新的充值凭证卡牌才能复活。

## 5. 🔮 知识提纯与固化 (Lore Synthesis & RAG)

这套系统的终极目标是建立系统性的认知优势，即 **RAG（Retrieval-Augmented Generation）与记忆提纯**。
- **初级合成**：把三张 `[零散研报片段]` 卡牌和一个 `[学术提炼模板]` 卡牌，同时塞进【分析】动词的插槽。转圈 10 分钟后，掉落一张高级的 `[行业全景综述]` 卡牌。
- **终极固化 (MEMORY.md)**：将高级洞察卡牌拖入【记忆】动词，这张卡片就会变成不会衰减、泛着金光的“永久知识实体”，固化到 `D:\openclaw-workspace\MEMORY.md` 数据库中。以后无论遇到什么新任务，你都可以随时从抽屉里把它掏出来，作为极其强大的高阶上下文（Context）放入插槽。
